## SYNC-2026-07-04b — ROW_SPLIT: real fix for the K≥48 baseline spill cliff
- **FP32 baseline still non-physical at K≥48** (K32 5.3×, K48 **36.7×**, K64 **37.3×** compute "speedup"). The 07-03d RR_C fix only reduced the spill (44×→30×), it did not remove it. *Root cause, quantified:* one block covers a whole XB=32-row tile-row, so each thread persistently holds `RPT·RR_C·RC_C = K²/32` accumulator floats (K48: 96, K64: 192), but `__launch_bounds__(1024,1)` caps threads at 64 regs. ptxas -v on the RUNNING templates: K32 ⟨8,2,2⟩ 56 B stack/thread (fits L1) vs K48 ⟨8,3,3⟩ **280 B** and K64 ⟨32,2,2⟩ **656 B** → 280–670 KB/SM working set ≫ 96 KB L1 → every per-tile accumulator flush is an L2/DRAM round-trip. Traffic estimate K48: ~0.57 MB/tile-visit × 1.45 M tiles ≈ **830 GB/iter** ≈ 2.3 s at 3060 bandwidth; measured 3.16 s/iter. On top: `yb_eff` 256→128 at K>32 doubles the tile count (2× more flushes). Baseline collapsed 230→63 (K48) / 51 (K64) GFlops/s.
- **Fix: ROW_SPLIT** — split each tile-row across `gridDim.y = G` sibling blocks (K16:1, K32:2, K48:4, K64/96:8); each block keeps the full 1024-thread geometry but owns only 32/G rows, so RPT drops G-fold and the running templates become ⟨2..4, RR, RC⟩ with **0–64 B stack** (ptxas-verified). Per-row arithmetic order unchanged → numerically identical up to existing atomicAdd nondeterminism (verified: ml100k K48 trajectories match to ~1e-6). Cost: only the cooperative smem feature staging repeats per sibling block (L2-cached); segment reads are per-row (not duplicated), total atomic output volume unchanged. Kernel: `fused_kernels.cuh` (`DZ = blockDim.z`, `row_base = blockIdx.y·RPT·DZ`); hosts: `ROW_SPLIT` define + `rows_per_thread = XB/(DZ·ROW_SPLIT)` + `dim3(blocks, ROW_SPLIT)` in both `.cu`. Override per-build with `-DROW_SPLIT=n`.
- **Consequence for the thesis:** headline speedups at K≥48 must be re-measured against the fixed baseline; expect them to land near the K16/K32 range (~4–6×) instead of 30–37×.

## SYNC-2026-07-04
- **Scalar kernel missing `const __restrict__`** — WMMA kernels already had `const __restrict__` on read-only pointers, but the scalar kernel (`compute_LHS_RHS_BALS_block`) — the spill bottleneck — used bare `int*`/`float*`.
